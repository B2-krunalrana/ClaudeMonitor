1

Who We Are

ClaudeMonitor is a free, open-source Chrome extension developed and maintained by Krunal Rana. The extension is published as an independent personal project and is not affiliated with Anthropic, Claude.ai, or any corporate entity.

The full source code is available at github.com/krunalrana/ClaudeMonitor — you are encouraged to read and audit it.

2

What Data We Collect

ClaudeMonitor collects only the minimum data required to show you accurate usage statistics:

  • 📊
    Token Usage Counts Approximate number of tokens consumed in your Claude.ai sessions during the current 5-hour window and weekly periods. This is calculated locally from intercepted API responses.
  • 🏢
    Organization ID Your Claude.ai organization ID (a non-personally-identifiable identifier). This is used to synchronize usage data across your devices via Firebase.
  • ⏱️
    Reset Timestamps The time at which your session or weekly usage limits are scheduled to reset.
  • 🌐
    UI Language Preference Your selected interface language (stored locally in browser storage only).
We do NOT collect: message content, conversation history, personal names, email addresses, IP addresses, browsing history, or any other personally identifiable information.
3

How We Use Your Data

  • 📈
    Display Usage Statistics Token counts and reset timers are displayed directly in the Claude.ai sidebar and popup.
  • 🔄
    Cross-Device Sync Your organization ID and usage data are synced via Firebase so your stats are consistent across multiple devices or browser profiles.
  • 🔔
    Reset Notifications (optional) If you enable it, we use stored reset timestamps to send a browser notification when your usage limit resets.
4

Data Storage & Third Parties

  • 💾
    Browser Local Storage Most data (language preference, notification settings, cached usage) is stored locally in your browser using the standard chrome.storage.local API. It never leaves your device unless explicitly synced.
  • 🔥
    Firebase (Google) — Sync only We use Firebase Realtime Database to sync your organization ID and usage totals across devices. Firebase's own privacy policy applies. No PII is transmitted — only your org ID and aggregate token counts.
  • 🚫
    No Analytics, No Ads, No Tracking We do not use any analytics platforms (Google Analytics, Mixpanel, etc.), advertising networks, or behavioral tracking of any kind.
5

Chrome Permissions Explained

Here is exactly why each permission is requested:

  • 🌐
    host_permissions: claude.ai Required to inject the usage UI into the Claude.ai page and intercept API responses to count tokens.
  • 🔌
    webRequest Used to intercept Claude.ai API responses (locally, never sent externally) to extract usage data.
  • 💿
    storage Saves your preferences and usage cache locally in your browser.
  • 🔔
    notifications Used only if you opt in to usage-reset notifications.
  • 🕐
    alarms Schedules background checks for usage reset times.
6

Data Retention & Deletion

Usage data (token counts, reset timestamps) is stored temporarily and automatically cleared when your usage period resets. Language and notification preferences are stored until you change or remove them.

To delete all data immediately: go to chrome://extensions → ClaudeMonitor → Remove extension. All locally stored data is erased. Firebase-synced data will expire automatically within 24 hours.

7

Children's Privacy

ClaudeMonitor is not directed at children under the age of 13. We do not knowingly collect any information from children. If you believe a child has provided data through this extension, please contact us immediately.

8

Changes to This Policy

Any material updates to this Privacy Policy will be announced via a new version release on the Chrome Web Store and via the GitHub repository. The "Last Updated" date at the top of this page will always reflect the most recent revision.
