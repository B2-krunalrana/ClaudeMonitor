Who We Are

ClaudeMonitor is a free, open-source Chrome extension developed and maintained by Krunal Rana. It is published as an independent personal project and is not affiliated with Anthropic, Claude.ai, or any corporate entity.

The complete source code is publicly available at github.com/b2-krunalrana/ClaudeMonitor . You are encouraged to read and audit it.

2

What Data We Collect

ClaudeMonitor collects the minimum data needed to display accurate usage statistics:

  • 📊
    Token Usage Counts Approximate tokens consumed during your Claude.ai sessions (current 5-hour window & weekly). Calculated locally from intercepted API responses.
  • 🏢
    Organization ID Your Claude.ai org ID — a non-personally-identifiable opaque string used to sync usage data across your devices via Firebase.
  • ⏱️
    Reset Timestamps The time at which your session or weekly usage limits are scheduled to reset.
  • 🌐
    UI Language Preference Your selected interface language, stored only in local browser storage.
🚫
We do NOT collect: message content, conversation history, names, email addresses, IP addresses, browsing history, or any personally identifiable information.
3

How We Use Your Data

  • 📈
    Display Usage Statistics Token counts and reset timers are rendered in the Claude.ai sidebar and extension popup.
  • 🔄
    Cross-Device Sync Your org ID and usage totals are synced via Firebase so your stats are consistent across devices.
  • 🔔
    Reset Notifications (opt-in only) If you enable this, stored reset timestamps trigger a browser notification when your limit resets.
4

Data Storage & Third Parties

  • 💾
    Browser Local Storage Preferences and cached usage data are stored using the chrome.storage.local API. Data never leaves your device unless explicitly synced.
  • 🔥
    Firebase (Google) — Sync only We use Firebase Realtime Database for cross-device sync of your org ID and token totals. Firebase's privacy policy applies. No PII is transmitted.
  • 🚫
    No Analytics · No Ads · No Tracking We use zero analytics platforms, no advertising networks, and no behavioral tracking.
5

Chrome Permissions Explained

Every permission is requested for a specific reason — here's exactly what each one does:

Permission Why it's needed
host_permissions
claude.ai		 Inject the usage UI into the Claude.ai page and intercept API responses locally to count tokens.
webRequest Intercept Claude.ai API responses (processed locally, never forwarded externally) to extract usage data.
storage Save your preferences (language, notification settings) and usage cache locally in your browser.
notifications Send an opt-in browser notification when your Claude usage limit resets.
alarms Schedule background checks for usage reset times.
tabs Communicate between the background service worker and active claude.ai tabs.
6

Data Retention & Deletion

Token counts and reset timestamps are stored temporarily and automatically cleared when your usage period resets. Language and notification preferences persist until you change them.

To delete everything immediately: open chrome://extensions, find ClaudeMonitor, and click Remove extension. All local data is erased instantly. Firebase-synced data expires automatically within 24 hours.

7

Children's Privacy

ClaudeMonitor is not directed at children under 13. We do not knowingly collect any information from children. If you believe a child has used this extension, please contact us immediately.

8

Changes to This Policy

Material updates will be announced via a new Chrome Web Store release and a GitHub commit. The "Updated" date at the top always reflects the latest revision.
